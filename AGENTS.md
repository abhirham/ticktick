# Agent Instructions

## Collaboration

- Spin up multiple agents when necessary to parallelize investigation, implementation, or verification work.
- Commit all checkpoints as work progresses, using clear commit messages for each checkpoint.

## UI Implementation

- Always follow [style-guide.md](style-guide.md) like a bible when designing or implementing UI in this project.
- Treat `style-guide.md` as the source of truth for layout, spacing, typography, colors, components, states, and visual hierarchy.
- Do not invent new UI patterns, colors, spacing, shadows, typography, or component styling unless `style-guide.md` explicitly leaves the detail unspecified.
- When a UI detail is unclear, choose the option that visually matches the mockups and documented tokens in `style-guide.md` most closely.
- Before finishing any UI work, compare the result against `style-guide.md` and revise anything that drifts from it.
